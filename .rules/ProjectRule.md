1. this project uses qlib environment, must run 'conda activate qlib' when a new terminal window created or before any new session
2. 如果调试期间发生多个错误，对于相关性不大的错误，绝对不能一次性尝试解决所有问题，必须逐个解决，进行测试，经过用户确认后，再尝试处理下一个错误。没有用户的许可，不能自行解决下一个错误。没有用户的明确许可，不能自己连续解决多个错误。绝对要遵循：每次只能处理一个错误。

3. 在每个问题解决完成后，都需要在 issues 目录下创建一个新的 issue 文件，记录问题的详细信息和解决方案。特别是问题处理过程中成功的步骤，应该详细记录下来，以便后续参考和学习。文件名称应该以问题编号命名，例如 0001-fix_chrono_header_issue.md， 数字部分应该从 0001 开始，不可重复，依次递增。
